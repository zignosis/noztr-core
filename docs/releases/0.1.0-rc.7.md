# noztr-core 0.1.0-rc.7

Date: 2026-03-27
Release type: breaking rc

## Summary

`0.1.0-rc.7` is the seventh public release candidate for `noztr-core`.

Relative to `0.1.0-rc.6`, this RC packages the remaining public route-local cleanup already on
`master`. Several routes now use shorter function families where the route already carries the
context, the internal crypto/bech32 support layout is tighter, and the docs are more precise about
determinism and allocator use. Wire behavior, trust-boundary posture, and kernel scope stay
otherwise unchanged.

## Changes Since `0.1.0-rc.6`

- shortened remaining route-local function families in:
  - `nip51_lists`
  - `nip66_relay_discovery`
  - `nip78_app_data`
  - `nip34_git`
  - `nip64_chess_pgn`
  - `nip11`
  - `nip44`
  - `nip59_wrap`
- tightened public docs so determinism and allocator expectations are more precise
- renamed the internal secp support surface from `adapter` to `boundary`
- shared internal bech32 core support across routes

## Breaking Changes

- shortened route-local function families in:
  - `nip51_lists`
  - `nip66_relay_discovery`
  - `nip78_app_data`
  - `nip34_git`
  - `nip64_chess_pgn`
  - `nip11`
  - `nip44`
  - `nip59_wrap`
- migration guide:
  - [../guides/migrating-from-0.1.0-rc.6.md](../guides/migrating-from-0.1.0-rc.6.md)

## Compatibility Notes

- Zig toolchain floor for this RC line remains `0.15.2`
- optional I6 exports remain build-flag gated
- wire formats, trust-boundary behavior, and kernel-vs-SDK scope remain unchanged by this RC
