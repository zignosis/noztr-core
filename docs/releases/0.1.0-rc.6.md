# noztr-core 0.1.0-rc.6

Date: 2026-03-25
Release type: breaking rc

## Summary

`0.1.0-rc.6` is the sixth public release candidate for `noztr-core`.

This RC continues the pre-`1.0` public-surface cleanup. Route-local names are shorter where the
route already carries the context, one overlapping `nip71` video shape is collapsed into a single
explicit kind, and the public docs routing is tighter. Wire behavior, trust-boundary posture, and
kernel scope stay otherwise unchanged.

## Public Highlights

- shortened route-local type, function, and error names across the highest-noise remaining routes
- made `nip71_video_events.Video` expose one explicit `kind: VideoKind`
- kept the `nip72` cleanup honest by restoring `AddressableTarget` where the shorter `Target`
  proved too ambiguous in-context
- tightened `examples/README.md` and the top-level `README.md` so public routing is clearer for
  both humans and LLMs
- fixed `zap_build_coordinate_tag` so the public builder maps coordinate validation into
  `InvalidCoordinateTag` correctly

## Breaking Changes

- shortened route-local public surface in:
  - `nip37_drafts`
  - `nip45_count`
  - `nip23_long_form`
  - `nip99_classified_listings`
  - `nip98_http_auth`
  - `nip72_moderated_communities`
  - `nip46_remote_signing`
  - `nip47_wallet_connect`
  - `nip54_wiki`
- changed `nip71_video_events.Video` from `flavor + addressable` to one `kind: VideoKind`
- changed `nip72_moderated_communities.Post` and `Approval` to use the clearer `Relation` and
  `ApprovedTarget` shapes
- migration guide:
  - [../guides/migrating-from-0.1.0-rc.5.md](../guides/migrating-from-0.1.0-rc.5.md)

## Compatibility Notes

- Zig toolchain floor for this RC line remains `0.15.2`
- optional I6 exports remain build-flag gated
- wire formats, trust-boundary behavior, and kernel-vs-SDK scope remain unchanged by this RC
