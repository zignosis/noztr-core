# noztr-core 0.1.0-rc.1

Date: 2026-03-21
Release type: rc

## Summary

`0.1.0-rc.1` is the first intentional public release candidate for `noztr-core`.

This RC establishes the documented protocol-kernel surface as a public release line, with public
docs/examples routing, a conservative pre-`1.0.0` compatibility posture, and the current bounded
Nostr helper surface available under one versioned tag.

It also includes the latest additive kernel work, most notably first-class native strict legacy
`NIP-04` direct-message support and tighter shared trust-boundary helper consistency with expanded
regression coverage.

## Public Highlights

- first intentional public RC for the documented `noztr-core` protocol-kernel surface
- public docs, examples, contributor routing, changelog, and versioning policy are now in place
- broad current surface includes bounded helpers for core events/filters, identity/auth,
  wallet/key material, legacy and modern private messaging, remote signing, wallet-connect, lists,
  groups, channels, communities, media metadata, and relay discovery
- a notable late-cycle additive kernel slice is first-class `noztr.nip04` support for legacy
  kind-4 DMs, including canonical local crypto, payload parse/serialize, strict DM event-shape
  validation, and interoperability for standard reply `e`-tag forms
- shared lower-hex, URL, relay-URL, and private-JSON helper consolidation with added regression
  coverage

## Breaking Changes

- none intended for this first public RC line
- the public line remains pre-`1.0.0`, so compatibility should still be treated conservatively

## Compatibility Notes

- Zig toolchain floor for this RC line is `0.15.2`
- `NIP-04` support is limited to strict legacy kind-4 DM handling
- deprecated `NIP-04` private-list compatibility remains out of scope
- `noztr-core` remains the protocol-kernel layer; workflow, transport, storage, and session
  composition still belong in `noztr-sdk`
