# noztr-core 0.1.0-rc.1

Date: 2026-03-21
Release type: rc

## Summary

`0.1.0-rc.1` is the first intentional public release candidate for `noztr-core`.

This RC establishes the documented protocol-kernel surface, adds first-class native strict legacy
`NIP-04` direct-message support, and tightens shared trust-boundary helpers behind more explicit
internal contracts and better regression coverage.

## Public Highlights

- first-class `noztr.nip04` support for legacy kind-4 DMs
- canonical local `NIP-04` encrypt/decrypt, payload parse/serialize, and strict DM event-shape
  validation
- improved interoperability for standard legacy DM reply `e`-tag forms
- shared lower-hex, URL, relay-URL, and private-JSON helper consolidation with added regression
  coverage

## Breaking Changes

- none intended for this first public RC line
- the public line remains pre-`1.0.0`, so compatibility should still be treated conservatively

## Compatibility Notes

- Zig toolchain floor for this RC line is `0.15.2`
- `NIP-04` support is limited to strict legacy kind-4 DM handling
- deprecated `NIP-04` private-list compatibility remains out of scope
- `noztr-core` remains the protocol-kernel layer; workflow, transport, storage, and session
  composition still belong in `noztr-sdk`

## Upgrade Guidance

- downstream consumers can now integrate legacy kind-4 DMs against `noztr.nip04`
