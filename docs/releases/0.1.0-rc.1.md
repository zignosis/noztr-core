# noztr-core 0.1.0-rc.1

Date: 2026-03-21
Release type: rc

## Summary

`0.1.0-rc.1` is the first intentional public release candidate for `noztr-core`.

It establishes a Zig-native Nostr protocol kernel with documented public routing, examples, and a
conservative pre-`1.0.0` compatibility posture. This RC is meant to be evaluated as the first
public versioned cut of the library as a whole, not as a single-feature release.

## What This Library Is

`noztr-core` is the deterministic protocol-kernel layer in the `noztr` ecosystem.

It is designed to provide:

- bounded parsing, validation, serialization, signing, verification, and reduction helpers
- caller-owned buffers and explicit typed trust-boundary errors
- deterministic protocol surfaces that higher-level SDKs and applications can build on

It is not trying to be:

- a relay runtime
- a websocket or TLS stack
- a storage layer
- a session, workflow, or application framework

Those broader runtime and composition responsibilities belong in `noztr-sdk` or in application
code above the kernel.

## Coverage Snapshot

This RC exposes a broad bounded surface across the current implemented line, including:

- core events, filters, and relay message grammar
- identity, relay info, auth, and HTTP-auth helpers
- wallet, mnemonic, key-encryption, and bounded entropy helpers
- legacy and modern private messaging helpers
- remote-signing, wallet-connect, and relay-admin typed contracts
- lists, groups, channels, communities, polls, media metadata, wiki, calendar, live-activity, and
  video metadata helpers
- relay-discovery, zap-goal, handler, nutzap, and AND-filter helpers

For the exhaustive implemented surface and split/optional module status, see
[docs/reference/nip-coverage.md](../reference/nip-coverage.md).

## Public Highlights

- first public RC tag for the documented `noztr-core` kernel surface
- public docs, examples, contributor routing, changelog, and versioning policy are now in place
- the public line now includes both legacy and modern bounded private-message surfaces
- shared lower-hex, URL, relay-URL, and private-JSON helper logic has been consolidated behind
  clearer internal contracts with stronger regression coverage

## Tradeoffs And Limitations

- this line remains pre-`1.0.0`; compatibility should still be read conservatively
- `noztr-core` prefers strict bounded trust-boundary behavior over maximally permissive parsing
- several surfaces are intentionally split with `noztr-sdk` rather than widened into workflow or
  runtime ownership

## Compatibility Notes

- Zig toolchain floor for this RC line is `0.15.2`
- optional I6 exports remain build-flag gated
- `noztr-core` and `noztr-sdk` are complementary layers, not competing ones
- some protocol areas are intentionally bounded to kernel-safe slices, with broader workflow and
  runtime policy left to `noztr-sdk` or application code

## Docs And Examples

- public docs now route through `README.md`, `docs/INDEX.md`, and `examples/README.md`
- the docs surface includes scope/tradeoff notes, intentional divergences, compatibility guidance,
  API reference, contract map, and NIP coverage
- examples and hostile examples are part of the public teaching surface, not just demos

## Verification

- `zig build test --summary all`
- `zig build`

## Adoption Guidance

- adopt this line as an RC, not as a long-established stable compatibility baseline
- start with the public routing docs instead of internal planning material
- use `noztr-core` when you want the deterministic kernel layer
- use `noztr-sdk` when you want higher-level runtime, transport, workflow, and application-facing
  composition on top of that kernel
